ADR-001
Title: OpenAPI-First API Design

Status:
Accepted

Decision:
All HTTP APIs shall be defined through OpenAPI specifications before implementation.

Rationale:
- Single source of truth
- Easier frontend integration
- Automatic documentation
- Contract validation
- Future SDK generation

Consequences:
- Every endpoint must exist in OpenAPI before implementation.
- Controllers must conform to specification.
- Shared DTOs derive from contracts.
